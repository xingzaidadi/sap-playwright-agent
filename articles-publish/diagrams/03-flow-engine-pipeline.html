Flow Engine：从 YAML 到可审计的执行结果

脚本解决一次执行，Flow Engine 解决长期维护
1

加载 YAML

解析 Flow 定义
提取 params、steps、metadata

2

参数校验

检查必填参数
类型匹配
缺参即停

3

Dry-run

不打开浏览器
预览流程路径
验证分支逻辑

4

执行 Steps

逐步调用 Action
on_error 策略
条件分支 / 子流程

5

Report

HTML 报告
截图 + Trace
步骤耗时 + 结果

脚本方式

await login()
await page.click('#btn')
await page.fill('#input', value)
// 失败了？翻控制台日志吧...

Flow 方式

name: goods-receipt
steps:
  - action: navigate_tcode
  - action: fill_fields
  - action: click_button
    on_error: screenshot_and_report
Flow = 可参数化 + 可组合 + 可 dry-run + 可报告 + 可复用
