Page Object 不是最高抽象

在通用企业 Web 自动化框架里，Page Object 有价值，但它应该待在 Adapter 内部。

容易退化的做法

Flow既知道业务顺序，也知道页面怎么点
↓
Page Object越来越大，变成 God Object
↓
Playwrightselector、iframe、键盘细节外泄
问题：一旦接入 OA / CRM / SRM，Flow 会被页面细节污染，Page Object 也会从页面封装退化成脚本垃圾桶。

更稳的边界

Flow Engine只编排业务步骤
↓
Action Registry把动作路由到系统能力
↓
Adapter系统接入边界，表达领域能力
↓
Page Object只封装 Adapter 内部页面细节
结论：Adapter-first, Page Object-inside。Flow 不暴露 selector，Page Object 不编排跨系统业务。
对外是 Adapter，对内用 Page Object。
