通用企业 Web 自动化框架：从 SAP 开始，但不止于 SAP

SAP WebGUI 是第一个高复杂度 Adapter，用来验证通用内核是否经得住真实后台系统的压力。
Core

通用内核

把所有系统都会复用的能力沉淀下来，避免每接一个后台系统就重写一套脚本基础设施。

Flow Engine流程编排、条件分支、错误策略
Action Registry把业务动作路由到系统能力
Report / Trace让执行过程可审计、可复盘
Adapter

系统适配层

SAP、OA、CRM、SRM 的差异不进入 Flow，而是收敛到各自 Adapter。

SAP Adapteriframe、TCode、readonly、系统消息
OA Adapter审批流、组织选择器、附件上传
CRM Adapter客户搜索、线索阶段、分页表格
Runtime

执行与 AI 辅助

AI 不直接接管浏览器，它负责意图解析、异常诊断和草稿生成。

Playwright稳定执行浏览器动作
AI Diagnose失败截图、日志、上下文分析
Skill约束 Agent 按架构产出
Flow Engine编排业务流程
Action Registry路由业务动作
Adapter隔离系统差异
Page Object封装页面细节
Playwright执行浏览器动作
RPA 是参照系，不是归属；SAP 是第一个样例，不是框架边界。
