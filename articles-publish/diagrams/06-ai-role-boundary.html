AI 在通用自动化框架里的正确位置

AI 负责理解和建议，Flow / Adapter / Runtime 负责执行

AI 做什么

理解、判断、生成、解释
  • 意图解析"帮我查 PO" → Flow + params
  • 异常诊断截图 + a11y tree + 错误日志 → 原因分析
  • 开发辅助从录制素材生成 Flow / Adapter 草稿
  • 交互追问参数缺失时向用户补问

确定性代码做什么

编排、执行、记录、校验
  • Flow Engine步骤编排、条件分支、on_error
  • Action Registry路由 action → adapter method
  • Adapter + Page Object页面操作、系统差异隔离
  • Report / Trace截图、步骤日志、HTML 报告

AI 不能绕过的边界

Flow Contract
Approval Gate
Promotion Gate
Evidence Report
