从 SAP 自动化到通用企业 Web 自动化框架

V1 到 V3 第一轮收口完成：从采集、可信执行到第二 Adapter 验证
Core 不是 SAP，SAP 是第一个 Adapter 样例
V1

采集与执行闭环

已完成

先做 Recording Pack，而不是完整录制回放。把 SOP、截图、selector 候选和等待证据沉淀成可 review 的自动化素材。

  • record-flow / compile-recording
  • YAML Flow + FlowRunner
  • dry-run 与参数校验
  • 截图证据与 HTML Report
V2

可信执行框架

已完成

从能跑升级到能证明、能阻断、能审计。企业后台自动化必须处理不可逆动作和系统边界。

  • Run Context + Step Evidence
  • 不可逆动作 Approval Gate
  • 只读 Flow 与变更 Flow 分离
  • SAP Adapter 不污染 Core
V3

多系统半自主

第一轮完成

不是继续堆 SAP Flow，而是用 SRM 作为第二 Adapter，证明同一套 Core 可以迁移到更多企业 Web 系统。

  • Action Registry
  • Adapter Registry
  • SRM query / create / confirm / generate
  • uploadPOScan 单独审查

框架公式

Recording Pack → Flow Engine → Action Registry → Adapter → Evidence + Approval

判断标准

不要用业务 Flow 数量判断成熟度。真正的成熟度来自框架机制是否能迁移到第二个系统。
