# Legacy AI Web Auto Drafts

这个目录存放 `articles/` 目录迁移过来的 4 篇早期文章草稿。

这些文档不是当前发布稿，也不是当前工程边界的权威说明。

## 为什么归档到 drafts

这些草稿写于项目早期，主线更偏向“AI 看屏幕后自主操作任意网页”。V3 收口后，项目正式路线已经更谨慎：

- Flow / Adapter / Runtime 负责确定性执行；
- AI 负责意图解析、异常诊断、开发辅助和 Skill 入口；
- 不可逆业务动作必须有 approval gate；
- SAP / SRM 真实操作不能被描述成无人值守自动执行；
- 不能承诺“任何网页”“任意系统”“自动化企业所有 Web 系统”。

因此，这 4 篇只作为旧选题、表达素材和历史路线参考，不放在 `articles-publish/` 根目录。

## 整理时做过的处理

- 从仓库根部 `articles/` 移入 `articles-publish/drafts/legacy-ai-web-auto/`；
- 给每篇开头补充“归档状态”；
- 将示例 URL、PO 号、物料凭证号替换为占位符；
- 保留原文主体，避免过度重写导致历史依据丢失。

## 后续使用方式

如果后续要从这些草稿中提炼文章，只建议抽取：

- 问题引入；
- a11y tree 技术解释；
- Skill 评审维度；
- 成本控制和运维监控表。

不建议复用：

- “AI 操作任何网页”；
- “一套框架自动化企业所有 Web 系统”；
- “不绑定系统”；
- “生产级”；
- 任何暗示 AI 可以默认执行不可逆 SAP/SRM 业务动作的段落。

当前正式发布材料优先看：

```text
articles-publish/00｜发布总目录.md
articles-publish/09｜从 V1 到 V3：我如何把 SAP 自动化项目收敛成通用企业 Web 自动化框架.md

```
